========================================================================
TradingAPIConnectionPoint.exe
========================================================================
Синглетон-сервис для подключения к единственному терминалу Quik.
Поскольку приложений-клиентов может быть несколько. А терминал поддерживает только одно подключение АПИ, 
сделано следующее решение:
Клиентсткое приложение запускает процесс синглетона с параметром - путь к терминалу Quik.

TradingAPIConnectionPoint.exe [path]
Если синглетон-сервис для данного пути уже поднят, то новый сервис не стартует.
Если синглетон-сервис для данного пути терминала Quik еще не существует, то поднимается новый сервис 
и устнавливает соединение.

Результаты запуска процесса:
1 - точка подключения уже существует. Нужно подключаться к поднятому процессу.
-1 - указанного пути к терминалу Quik не существует.
0 - процесс успешно поднялся и завершился без ошибок.
